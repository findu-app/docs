## Authentication

All FindU APIs use JWT tokens from Supabase Auth:

```bash
Authorization: Bearer YOUR_JWT_TOKEN
```

### Obtaining Tokens

#### Web Applications
```javascript
import { supabase } from '@/lib/supabase'

// Get current session
const { data: { session } } = await supabase.auth.getSession()
const token = session?.access_token

// Use in API calls
fetch('https://api.findu.app/endpoint', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
})
```

#### iOS Applications
```swift
import Supabase

// Get current session
let session = try await supabase.auth.session
let token = session.accessToken

// Use in API calls
var request = URLRequest(url: url)
request.setValue("Bearer \(token)", forHTTPHeaderField: "Authorization")
```

## Common Headers

```bash
Content-Type: application/json
Authorization: Bearer YOUR_JWT_TOKEN
X-Client-Version: 1.0.0
```

## Response Format

All API responses follow a consistent format:

### Success Response
```json
{
  "data": {
    // Response data
  },
  "metadata": {
    "timestamp": "2024-01-15T10:30:00Z",
    "version": "1.0.0"
  }
}
```

### Error Response
```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error message",
    "details": {
      // Additional error context
    }
  }
}
```

## Rate Limiting

API rate limits vary by endpoint type:

| Endpoint Type | Limit |
|---------------|-------|
| Read operations | 300/minute |
| Write operations | 60/minute |
| Analytics | 100/minute |
| Batch operations | 20/minute |

Rate limit headers:
```
X-RateLimit-Limit: 300
X-RateLimit-Remaining: 250
X-RateLimit-Reset: 1640995200
```

## Pagination

List endpoints support pagination:

```bash
GET /api/students?limit=20&offset=40
```

Response includes pagination metadata:
```json
{
  "data": [...],
  "pagination": {
    "total": 150,
    "limit": 20,
    "offset": 40,
    "has_more": true
  }
}
```

## Versioning

API versions are specified in the URL path:
- Current: `/v2/endpoint`
- Legacy: `/v1/endpoint` (deprecated)
